# Home Automation Project Prompt Customization

## Context
You are working with a home automation project that integrates:
- **Alexa voice control** via Sinric Pro (secure WebSocket)
- **UniFi WiFi management** via REST API
- **Python FastAPI microservice** running in Docker on Synology NAS

## Default Behaviors

### When working with code
- Assume Python code follows FastAPI best practices
- Check for proper async/await patterns in API handlers
- Verify environment variable usage instead of hardcoded values
- Consider the Docker containerization and how services communicate

### When working with configuration
- Be aware of `.env` files that contain secrets (never display them)
- Understand Docker Compose networking and service dependencies
- Check health checks and restart policies in docker-compose.yml

### When analyzing errors
- Consider the three-layer architecture: voice interface → middleware → network control
- Check WebSocket connectivity between Sinric Pro and the service
- Verify UniFi API authentication and certificate validation issues
- Review Docker container logs for runtime errors

## File Type Defaults

### Python Files (main.py, FastAPI services)
- Use type hints for function parameters and returns
- Include fastapi decorators with proper HTTP method routing
- Handle both Sinric Pro events and UniFi API responses
- Include error handling for network timeouts

### Docker Files
- Base images should be minimal (python:3.11-slim)
- Include health checks for service availability
- Expose only necessary ports
- Use multi-stage builds for optimization

### Configuration Files (.env, docker-compose.yml)
- Support local development and production configurations
- Document all required environment variables
- Include commented examples for sensitive values

## Preferred Agent Usage

- **Use Code Reviewer** when: analyzing existing code, designing new features, suggesting improvements
- **Use Debugger** when: troubleshooting errors, investigating failures, analyzing logs
- **Use default agent** when: general questions, architecture decisions, documentation

## Additional Context
- The system must handle WebSocket disconnections gracefully
- UniFi API requires proper certificate handling (HTTPS)
- Sinric Pro uses HMAC-SHA256 signing for all messages
- All components should be containerized and version-controlled (except .env)
