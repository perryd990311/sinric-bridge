# Home Automation Workspace Instructions

## Overview
This workspace contains a home automation system integrating Sinric Pro (voice control) with UniFi WiFi and Python microservices running in Docker.

## Technology Stack
- **Frontend**: Alexa voice control via Sinric Pro
- **Backend**: Python FastAPI service
- **Infrastructure**: Docker, Docker Compose on Synology NAS
- **Network**: UniFi WiFi management via API
- **Documentation**: Markdown guides and API response examples

## Project Structure
- `alexa/` - Alexa integration and routine documentation
- `docker/` - Docker Compose configuration and environment setup
- `python-service/` - FastAPI application, requirements, Dockerfile
- `unifi/` - UniFi API documentation and setup guides
- `docs/` - Technical documentation and API responses

## Code Review Standards
When reviewing code, focus on:
1. **Security**: No API keys/secrets in code, use environment variables
2. **API Integration**: Proper error handling for UniFi and Sinric Pro APIs
3. **Docker**: Proper containerization, health checks, environment variable usage
4. **Python**: Type hints, error handling, FastAPI best practices
5. **Documentation**: Clear docstrings and inline comments for complex logic

## Architectural Patterns
The system follows an event-driven architecture:
```
Voice Command → Sinric Pro WebSocket → FastAPI Service → UniFi API → Network Control
```

## Development Guidelines
- All API keys must be in `.env`, never hardcoded
- Use Docker Compose for local testing
- Test UniFi API calls before integrating into service logic
- Keep FastAPI endpoints focused and modular
- Document API responses in `docs/` folder
